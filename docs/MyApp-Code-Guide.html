1. Project Overview

MyApp is a daily habit tracker built with React Native and Expo. Users can:

  • View today's habits in a scrollable card list
  • Check off habits and see a live progress bar
  • Track per-habit streaks (days in a row completed)
  • Add new habits via a bottom-sheet modal
  • Delete habits with a long-press confirmation

The app uses Expo Router for file-based navigation and all state is managed locally with React's useState hook — no external state library is needed.

4. App Entry — _layout.tsx

export default function RootLayout() {
  return (
    <Stack>
      <Stack.Screen name="index" options={{ headerShown: false }} />
    </Stack>

How Expo Router works

Every file inside app/ becomes a screen. The special file _layout.tsx defines the navigator that wraps those screens — think of it as the picture frame around the pictures.

5.2 Type Definition

type Habit = {
  id: string;
  name: string;
  completed: boolean;
  streak: number;
};

This TypeScript type describes the shape of a single habit. Every habit in the app must have exactly these four fields. TypeScript will show an error at compile time if any code tries to create a habit with missing or wrong-typed fields.

5.5 Event Handlers

toggleHabit — called on tap

function toggleHabit(id: string) {
  setHabits((prev) =>
    prev.map((h) => {
      if (h.id !== id) return h;
      const completing = !h.completed;
      return {
        ...h,
        completed: completing,
        streak: completing ? h.streak + 1 : Math.max(0, h.streak - 1),
      };
    })
  );
}
  • prev.map(...) creates a new array — we never mutate state directly in React
  • if (h.id !== id) return h — all other habits pass through unchanged
  • ...h spreads existing fields, then we override only completed and streak
  • Math.max(0, h.streak - 1) — streak never goes below zero
  • The functional form setHabits((prev) => ...) avoids stale closure bugs

addHabit — called on modal confirm

function addHabit() {
  const name = newHabitName.trim();
  if (!name) return;
  setHabits((prev) => [
    ...prev,
    { id: Date.now().toString(), name, completed: false, streak: 0 },
  ]);
  setNewHabitName('');
  setModalVisible(false);
}
  • .trim() removes accidental whitespace
  • if (!name) return guards against empty submissions
  • Date.now().toString() generates a unique ID from the current timestamp in ms
  • [...prev, newHabit] appends without mutating the existing array

deleteHabit — called on long-press

function deleteHabit(id: string) {
  Alert.alert('Delete Habit', 'Remove this habit?', [
    { text: 'Cancel', style: 'cancel' },
    {
      text: 'Delete',
      style: 'destructive',
      onPress: () => setHabits((prev) => prev.filter((h) => h.id !== id)),
    },
  ]);
}
  • Alert.alert renders a native OS dialog on both iOS and Android
  • style: 'destructive' renders the Delete button in red on iOS
  • prev.filter((h) => h.id !== id) returns a new array with the target removed

Two module-level colour constants drive the whole theme. Changing PRIMARY updates every button, checkbox, progress bar, and accent app-wide.

Why StyleSheet.create()?
Validates properties at load timeTypos like backgrondColor throw immediately
Sends styles to native layer onceReferences them by ID on every render — faster than re-serialising objects

Cross-platform shadows

...Platform.select({
  ios:     { shadowColor: '#000', shadowOpacity: 0.06,
             shadowRadius: 8, shadowOffset: { width: 0, height: 4 } },
  android: { elevation: 4 },
}),

iOS and Android use completely different shadow APIs. Platform.select returns the right object at runtime and the spread merges it into the style. On the web, both are ignored gracefully.

7. Hooks — useColorScheme.ts

export { useColorScheme } from 'react-native';

This one line re-exports React Native's built-in useColorScheme hook unchanged. The reason it exists as its own file is extensibility: if you later want to let users manually override the system theme, you only change this one file. Every component that imports from @/hooks/useColorScheme automatically gets the new behaviour — no component changes needed.

useColorScheme returns 'light', 'dark', or null and re-renders any component that uses it whenever the system appearance changes.

8. Scripts — reset-project.js

const fs   = require('fs');
const path = require('path');

const appDir     = path.join(__dirname, '..', 'app');
const indexFile  = path.join(appDir, 'index.tsx');
const layoutFile = path.join(appDir, '_layout.tsx');

// Delete every file in app/
fs.readdirSync(appDir).forEach((file) => {
  fs.rmSync(path.join(appDir, file), { recursive: true, force: true });
});

// Re-write blank index and layout
fs.writeFileSync(indexFile,  indexContent);
fs.writeFileSync(layoutFile, layoutContent);

console.log('Project reset to blank state.');

A developer utility that wipes the app/ directory back to a minimal blank state. Never shipped to users — run manually when starting fresh.

Run it

node scripts/reset-project.js
APIWhat it does

fs.readdirSync(appDir)Lists all files/folders in app/ synchronously
fs.rmSync(..., { recursive: true, force: true })Deletes a file or folder including its contents; force suppresses errors if missing

9. How to Run

Start the development server

cd "Example Project/MyApp"
npx expo start

Test on your phone

  1. Install Expo Go from the App Store (iOS) or Play Store (Android)
  2. Ensure your phone and computer are on the same Wi-Fi network
  3. Scan the QR code — with the Camera app on iOS, or the Expo Go scanner on Android
